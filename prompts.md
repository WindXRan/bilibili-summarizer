# 洗稿 & 切分 Prompts

## 一、洗稿 Prompt

用在 ChatGPT / Claude 等对话式 AI 中，对提取的原始文案进行改写。

**Input:** `{asr_提取的文案}`

```
你是一个短视频脚本编剧。请将下面的文稿改写成适合竖屏短视频的口播脚本。

要求：
- 目标受众：手机用户，前3秒必须抓住注意力
- 一句一个画面：每句都要能在脑海中形成独立画面
- 去掉啰嗦的铺垫和过渡词，直接进主题
- 修复语音识别错误，语句通顺自然
- 用口语化短句，便于配音
- 每句一行，句与句之间空行分隔

输出格式：
[第一句]
[空行]
[第二句]
[空行]
[第三句]
...

文稿：
{asr_提取的文案}
```

**Output 示例:**
```
很多作者总觉得自己的文笔不好，写不出想要的东西。

但网文对好文笔的定义，跟你们想的完全是两码事。

今天我说明白：什么样的文笔才算合格？

四个标准，哪怕你只是小学生水平，掌握了也能让读者欲罢不能。

一、精准度。

很多人把辞藻堆砌当成好文笔。

写美女就闭月羞花沉鱼落雁，连衣服褶皱都要三个比喻句。

这种在网文里不但不合格，反而是累赘。

网文的核心是精准——用最简单的词，把读者拉进情境。

比如写一个人穷。

不用写他多凄惨，只需写：牙膏管已经被擀面杖压得像一张薄纸，但他还是试图用指甲掐出最后那一点白痕。

这一句，比他哭穷一百遍都高级。

所以第一步：删掉多余的形容词，看句子还能不能说清楚事。

能说清，你的文笔就落地了。
```

## 二、文本切分 Prompt

对洗稿后的文案进行时长规划，每一段对应一个画面。

**Input:** `{洗稿后的文案}`

```
你是一个短视频剪辑师。请将下面的脚本文案切分成视频片段，并为每段估算时长。

规则：
- 每段对应视频中的一个画面/字幕卡
- 按语义切分，不要断句
- 中文阅读速度按 4字/秒 估算
- 每段最短2秒，最长8秒
- 手机屏幕每段不超过20个字为佳，超过20字需切分
- 输出 JSON，可直接被程序解析

输出格式：
{
  "total_estimated_seconds": 总时长,
  "scenes": [
    {
      "id": 1,
      "text": "第一句文案",
      "estimated_seconds": 2.5,
      "style_hint": "画面建议（可选）"
    }
  ]
}

脚本：
{洗稿后的文案}
```

**Output 示例:**
```json
{
  "total_estimated_seconds": 92,
  "scenes": [
    {"id": 1, "text": "很多作者总觉得自己的文笔不好，写不出想要的东西。", "estimated_seconds": 3.5},
    {"id": 2, "text": "但网文对好文笔的定义，跟你们想的完全是两码事。", "estimated_seconds": 3.0},
    {"id": 3, "text": "今天我说明白：什么样的文笔才算合格？", "estimated_seconds": 2.5},
    {"id": 4, "text": "四个标准，哪怕你只是小学生水平，掌握了也能让读者欲罢不能。", "estimated_seconds": 4.0}
  ]
}
```

## 使用流程

```
1. main.py --asr       →  project/作者/标题_asr.txt    ← 原始提取
2. 用 洗稿Prompt + AI  →  project/作者/标题_洗稿.txt   ← 每句一行
3. 用 切分Prompt + AI  →  project/作者/标题_分镜.json   ← 时间规划
4. render.py            →  project/作者/标题.mp4        ← 合成视频
```
